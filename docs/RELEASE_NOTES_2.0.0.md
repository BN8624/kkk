# 세 왕관의 섬 2.0.0 릴리스 노트 초안

## 2.0 핵심 기능

- 화면별 컨트롤러와 취소 가능한 비동기 수명주기로 앱 구조를 정리했다.
- 외부 시나리오·리플레이를 크기·깊이·구조·의미·호환성 단계로 안전하게 검사한다.
- 리플레이 기반 로컬 플레이테스트 분석, 코칭, JSON·Markdown·CSV 보고서를 제공한다.
- 캠페인 평가 정책 5종과 품질 매트릭스, 시나리오 품질 검사, 공식 전장 6개를 추가했다.
- 한국어·영어 UI, 키보드·모달·스크린리더 상태, reduced motion을 지원한다.
- 설치 가능한 PWA, 오프라인 플레이, 사용자 선택 업데이트, 전체·선택 백업과 병합·교체 복구를 제공한다.

## 호환성

- 기존 1.5 저장 데이터와 v1·v2·v3 저장 마이그레이션을 유지한다.
- 1.5.x 및 2.0.x 리플레이는 동일 전투 규칙으로 정확 재생한다.
- 변환 가능한 과거 문서는 안전하게 마이그레이션하고, 검증되지 않은 버전은 경고한다.
- 지원하지 않거나 미래 버전인 문서는 원본 데이터를 바꾸지 않고 거부한다.

## 설치와 오프라인

지원 브라우저에서 공개 Pages 주소를 연 뒤 홈 화면 추가 또는 앱 설치를 선택한다. 한 번 온라인으로 연 다음에는 빠른 전투·캠페인·일일 도전·제작실·리플레이·공식 전장을 오프라인에서도 사용할 수 있다. 업데이트는 게임 도중 강제 적용되지 않으며 타이틀에서 사용자가 선택한다.

## 공식 시나리오 팩

초단기 전투, 20×20 대형 전장, 두 세력 결투, 중앙 요새 방어, 경제 목표, 특정 유닛 호위의 공식 전장 6개를 제공한다. 공식 원본은 읽기 전용이며 복제한 뒤 편집할 수 있다.

## 알려진 한계

- 데이터와 분석은 브라우저 로컬에만 저장되며 기기 간 자동 동기화하지 않는다.
- Canvas 지도의 모든 타일을 완전한 스크린리더 조작 대상으로 제공하지는 않지만 현재 턴·목표·선택 유닛·가능 행동을 DOM 상태로 알린다.
- 캠페인 품질 수치는 자동 평가 정책 결과이며 인간 플레이 검증으로 간주하지 않는다.
- 실제 GitHub Release 생성과 itch.io 업로드는 배포 권한 보유자가 생성된 산출물을 사용해 수행해야 한다.
- Playwright WebKit의 오프라인 에뮬레이션은 브라우저 내부 오류로 자동화할 수 없어 Chromium 모바일·PC에서 자동 검증하고 실제 iPhone은 수동 확인한다.
- 물리적 iPhone Safari 검증은 별도 수동 체크리스트가 필요하다.

## 업로드 절차

1. `npm ci && npm run build && npm run release:assets`를 실행한다.
2. `release/SHA256SUMS.txt`로 파일 무결성을 확인한다.
3. `release/three-crowns-island-2.0.0-itch.zip`을 itch.io HTML 게임으로 업로드한다.
4. 나머지 `release/` 파일을 GitHub Release 2.0.0 자산으로 첨부한다.
5. Actions의 모든 필수 작업과 Pages 배포 성공을 확인한다.
