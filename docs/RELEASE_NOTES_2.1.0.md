# 세 왕관의 섬 2.1.0 릴리스 노트 초안

## 2.1 핵심 변경

- **왕관의 심장 공정성 수리**: 중앙 왕관 위치를 실제 도착 턴 기준으로 공정하게 고르고(최소 도착·격차 제약), **3라운드에 활성화**되며 활성화 뒤 **4턴 연속 확보**로 승리한다. 왕관 인접에 적이 있고 주둔이 없으면 그 라운드는 **경합**으로 확보가 진행되지 않는다. AI는 왕관 확보·저지를 인식한다.
- 한 줄 규칙: `3라운드에 열리는 중앙 왕관을 4턴 연속 확보하면 승리 (경합 시 정지)` / `Opens on round 3 — hold the central Crown for 4 straight turns to win (contested rounds pause)`.
- 공개 설명·i18n·HTML 메타데이터를 2.1 제품 범위(9개 미션 캠페인, 모바일 전장 제작실, 결정론적 리플레이, 플레이 분석, 오프라인 PWA, 한국어·영어)에 맞게 갱신한다.

## 호환성

- 기존 1.5 저장 데이터와 v1·v2·v3 저장 마이그레이션을 유지한다.
- **리플레이**
  - 1.5.x · 2.0.x **비-crown** 시나리오: 동일 전투 규칙으로 **exact** 재생.
  - 2.0.x **crown-heart**: 왕관 규칙 변경으로 **playable-unverified**(재생 전용, exact 검증 없음).
  - 2.1.x: **exact**.
- **진행 중 구 crown-heart 저장**(구 왕관 규칙, 저장 형식 v3 이하 crown-hold 진행 중): 규칙 불일치로 **안전 폐기**(로드 시 null). 완료된 판·비-crown 저장은 마이그레이션 후 유지한다.
- 변환 가능한 과거 문서는 안전하게 마이그레이션하고, 검증되지 않은 버전은 경고한다.
- 지원하지 않거나 미래 버전인 문서는 원본 데이터를 바꾸지 않고 거부한다.

아래는 인간 플레이 재미 증명이 아니라 **자동 대체 정책 시뮬레이션**과 **구조적 도착 분석** 산출물이다.

- 집중 시뮬: `artifacts/crown-heart-balance-summary.md` (평가 정책×난이도×시드 통계)
- 도착 공정성 구조 분석: `artifacts/crown-heart-human-regression.md` (구/신 왕관 위치·도착 턴 비교; 과거 리플레이 재생이 아님)
- 시나리오 감사·공식 팩·결정론 등 기존 자동 검증 산출물(`artifacts/`)

## 설치와 오프라인

지원 브라우저에서 공개 Pages 주소를 연 뒤 홈 화면 추가 또는 앱 설치를 선택한다. 한 번 온라인으로 연 다음에는 빠른 전투·캠페인·일일 도전·제작실·리플레이·공식 전장을 오프라인에서도 사용할 수 있다. 업데이트는 게임 도중 강제 적용되지 않으며 타이틀에서 사용자가 선택한다.

## 알려진 한계

- 데이터와 분석은 브라우저 로컬에만 저장되며 기기 간 자동 동기화하지 않는다.
- Canvas 지도의 모든 타일을 완전한 스크린리더 조작 대상으로 제공하지는 않지만 현재 턴·목표·선택 유닛·가능 행동을 DOM 상태로 알린다.
- 캠페인·crown-heart 품질 수치는 자동 평가 정책 결과이며 인간 플레이 검증으로 간주하지 않는다.
- 실제 GitHub Release 생성과 itch.io 업로드는 배포 권한 보유자가 생성된 산출물을 사용해 수행해야 한다.
- Playwright WebKit의 오프라인 에뮬레이션은 브라우저 내부 오류로 자동화할 수 없어 Chromium 모바일·PC에서 자동 검증하고 실제 iPhone은 수동 확인이 필요하다(`docs/IPHONE_CHECKLIST.md`). 이 릴리스 노트는 물리 iPhone 실기기 검증 완료를 주장하지 않는다.

## 업로드 절차

1. `npm ci && npm run build && npm run release:assets`를 실행한다.
2. `release/SHA256SUMS.txt`로 파일 무결성을 확인한다.
3. `release/three-crowns-island-2.1.0-itch.zip`을 itch.io HTML 게임으로 업로드한다.
4. 나머지 `release/` 파일을 GitHub Release 2.1.0 자산으로 첨부한다.
5. Actions의 모든 필수 작업과 Pages 배포 성공을 확인한다.
