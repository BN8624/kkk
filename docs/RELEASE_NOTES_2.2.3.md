# 세 왕관의 섬 2.2.3 릴리스 노트

이슈 #17 외부 리뷰 후속 수리. **전술 게임 밸런스 검증 정직화 · 저장 신뢰성 · 연출·입력 결함**을 고친다.

자동 시뮬레이션 PASS는 종료 가능성·불법 상태·극단적 편중·난이도 순서를 뜻한다. 인간 플레이의 재미·가독성·조작감은 별도 실기 검증 대상이며 이 버전 노트에서 증명하지 않는다. 물리 iPhone 전체 체크리스트는 아직 완료되지 않았다.

## 수정 요약

### 밸런스 시뮬레이션 정본화

- 최종 6병종(`infantry`·`archer`·`cavalry`·`guardian`·`raider`·`crossbow`)을 `UNIT_TYPE_IDS` 정본으로 집계한다.
- 왕국 전체 승률 25–45%, 최고·최저 차 15%p 이하 / 시나리오별 20–50%, 차 25%p 이하 게이트.
- 정복 시나리오(`three-crowns`·`broken-strait`) 턴 제한 종료 ≤70%.
- 병종 편중: 단일 병종 ≤60%, 공용 기병 ≥8%, 고유 병종 적격 생산 40–95%.
- AI 수도 우선·교리·수도 배치·수호 태세 수치를 조정해 게이트를 실제 수치로 통과한다.

### 자동 저장 실패 가시화

- `saveGame` / `saveRaw` / `saveSettings`가 성공 여부 `boolean`을 반환한다.
- 일반 게임 저장 실패 시 세션 최초 1회 경고(한/영).
- 데이터 관리 화면에 기기 저장소 사용 가능/불가 표시(프라이빗 모드 단정 없음).

### 쇠뇌대 공격 연출

- 궁병: 화살 투사체 / 쇠뇌대: 더 빠르고 묵직한 볼트 투사체.
- 근접 병종은 돌진 유지. 공용 `attackAnimKind` 판정.

### 리사이즈·시나리오 좌표

- `fitCameraToTiles`가 자신 콜백만 교체·정리한다(`scale.off('resize')` 전체 제거 금지).
- 가져온 시나리오 좌표를 유한 정수로 검증하고 오류 경로를 포함한다.

### 문서 정직화

- README: 모바일 우선 설계·자동 테스트·iPhone Safari 구현 vs 물리 체크리스트 미완료를 구분.
- 자동 시뮬레이션 한계를 명시. 깨진 워크플로 한글 주석을 UTF-8로 복구.

## 설치와 오프라인

PWA 셸 캐시는 `three-crowns-shell-v2.2.3`으로 갱신된다. 업데이트는 게임 도중 강제 적용되지 않으며 타이틀에서 사용자가 선택한다.

## 업로드 절차

1. `npm ci && npm run build && npm run release:assets`를 실행한다.
2. `release/SHA256SUMS.txt`로 파일 무결성을 확인한다.
3. `release/three-crowns-island-2.2.3-itch.zip`을 itch.io HTML 게임으로 업로드한다.
4. 나머지 `release/` 파일을 GitHub Release 2.2.3 자산으로 첨부한다.
5. Actions의 모든 필수 작업과 Pages 배포 성공을 확인한다.
