# 세 왕관의 섬 2.2.3 릴리스 노트

이슈 #17 분할 마무리(#22 게이트 정본 + #27 밸런스 재설계). **전술 밸런스 조정 · 정복 턴 제한·기병 비중 게이트 통과 · 캠페인 회귀 방지 · 리플레이 호환 정직화**를 고친다.

자동 시뮬레이션 PASS는 종료 가능성·불법 상태·극단적 편중·난이도 순서·#22 게이트를 뜻한다. 인간 플레이의 재미·가독성·조작감은 별도 실기 검증 대상이며 이 버전 노트에서 증명되지 않는다. 물리 iPhone 전체 체크리스트는 아직 완료되지 않았다.

## 수정 요약

### 밸런스 시뮬레이션 정본 게이트 실제 조정

- 최종 6병종(`infantry`·`archer`·`cavalry`·`guardian`·`raider`·`crossbow`)을 `UNIT_TYPE_IDS` 정본으로 집계한다(#22).
- 왕국 전체 승률 25–45%, 최고·최저 차 15%p 이하 / 시나리오별 20–50%, 차 25%p 이하 게이트.
- 정복 시나리오(`three-crowns`·`broken-strait`) 턴 제한 종료 ≤70%.
- 병종 편중: 단일 병종 ≤60%, 공용 기병 ≥8%, 고유 병종 적격 생산 40–95%.
- **azure 시작 금은 40 유지**(#23 회귀 원인인 대폭 너프를 쓰지 않음). 수도 방어 +2·수호 태세 +2 유지.
- 다중 레버로 게이트를 맞춤: 정복 AI 수도 우선·마을 억제, 정복 수도 배치 근접, 기병 생산·비용, 자원 시작 기병·경제 소폭, 왕관 방어 완화, crown-heart 초반 전멸 유예.
- `audit:scenarios` 왕관의 심장 4턴 이하 종료 강제 판정을 해소한다. 정복 시나리오 조기 종료 경고는 수치와 함께 존재할 수 있다.

### 리플레이 호환

- 2.2.0–2.2.2: 2.2.3 밸런스·종료 규칙 재조정으로 결과가 달라질 수 있어 **playable-unverified**(재생만 허용, exact 검증 금지).
- 2.2.3+: 현행 exact.
- 2.1.x·1.5.x·2.0.x(비-crown) 등급은 유지. 2.0.x crown-heart는 기존처럼 playable-unverified.

## 설치와 오프라인

PWA 셸 캐시는 `three-crowns-shell-v2.2.3`으로 갱신된다. 업데이트는 게임 도중 강제 적용되지 않으며 타이틀에서 사용자가 선택한다.

## 업로드 절차

1. `npm ci && npm run build && npm run release:assets`를 실행한다.
2. `release/SHA256SUMS.txt`로 파일 무결성을 확인한다.
3. `release/three-crowns-island-2.2.3-itch.zip`을 itch.io HTML 게임으로 업로드한다.
4. 나머지 `release/` 파일을 GitHub Release 2.2.3 자산으로 첨부한다.
5. Actions의 모든 필수 작업과 Pages 배포 성공을 확인한다.
